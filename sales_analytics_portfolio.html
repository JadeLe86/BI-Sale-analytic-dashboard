SECTION 5 OF 8

DAX Measures

Production-ready Power BI DAX formulas with explanation for each measure. Organize these in a dedicated _Measures table for clean model hygiene.

01 — Total Revenue
Total Revenue = SUM( FactSales[SalesAmount] ) -- Aggregates all transaction amounts in the current filter context. -- Responds to all slicers: date, region, customer, product.
02 — Total Profit
Total Profit = SUM( FactSales[Profit] ) -- Profit is pre-calculated in the Fact table as (SalesAmount - CostAmount). -- Alternatively: SUM(SalesAmount) - SUM(CostAmount)
03 — Profit Margin %
Profit Margin % = DIVIDE( [Total Profit], [Total Revenue], 0 ) -- DIVIDE safely handles division by zero (returns 0 instead of error). -- Format as percentage in Power BI model properties.
04 — Total Orders
Total Orders = COUNTROWS( FactSales ) -- Counts every row (transaction) in the Fact table. -- Use DISTINCTCOUNT(SalesID) if rows may be duplicated at source.
05 — Total Customers
Total Customers = DISTINCTCOUNT( FactSales[CustomerKey] ) -- Counts unique customer IDs, not order rows. -- Essential: one customer may have many orders.
06 — Revenue Growth %
Revenue Growth % = VAR RevCY = [Total Revenue] VAR RevPY = CALCULATE( [Total Revenue], SAMEPERIODLASTYEAR( DimDate[FullDate] ) ) RETURN DIVIDE( RevCY - RevPY, RevPY, BLANK() ) -- SAMEPERIODLASTYEAR requires a marked date table. -- Returns BLANK() if no prior-year data exists (avoids misleading 0%).
07 — Revenue YTD
Revenue YTD = TOTALYTD( [Total Revenue], DimDate[FullDate] ) -- Accumulates revenue from Jan 1 to the last date in the filter context. -- Add a fiscal year end date argument if using non-calendar fiscal year.
08 — Revenue MTD
Revenue MTD = TOTALMTD( [Total Revenue], DimDate[FullDate] ) -- Accumulates revenue from the 1st of the current month to the latest date. -- Useful in the Executive Overview KPI cards for in-month tracking.
09 — Running Total Revenue
Running Total Revenue = CALCULATE( [Total Revenue], FILTER( ALL( DimDate ), DimDate[FullDate] <= MAX( DimDate[FullDate] ) ) ) -- Cumulative sum from the earliest date to the current row date. -- Used in area / waterfall charts showing cumulative performance.
10 — Average Order Value
Average Order Value = DIVIDE( [Total Revenue], [Total Orders], 0 ) -- Revenue per transaction. Rising AOV = higher transaction quality. -- Segment by product category for pricing optimization analysis.

Portfolio Case Study

Portfolio Project · Business Intelligence

Executive Sales Analytics Dashboard

A full Power BI solution transforming manual Excel reporting into a real-time executive intelligence system for a mid-market SME with $5M+ annual revenue.

Power BI SQL Server Excel DAX Star Schema
Business Problem

The client operated a growing distribution business across 5 regions with 1,200+ customers. Their reporting was entirely Excel-based: a finance analyst spent 3–4 days each month manually consolidating data from 6 different spreadsheets into a static PDF report. By the time leadership received it, the data was already 3 weeks old. No mechanism existed to answer ad-hoc questions — every data request required another manual extraction. Sales leaders had no visibility into regional performance gaps until month-end, meaning corrective actions were consistently too late to impact the period.

Solution Delivered

Designed and deployed a 4-page Power BI dashboard backed by a production-grade star schema data model. Consolidated all 6 source Excel files into a unified SQL staging layer with daily automated refresh. Built a governed KPI framework of 11 core measures using DAX with time-intelligence functions. Delivered an Executive Overview, Customer Analytics, Product Analytics, and Regional Performance page — each designed for a specific management audience.

Lessons Learned
  • Data quality issues consumed ~40% of project time — always build a data quality audit phase into the plan.
  • Executive stakeholders need onboarding alongside delivery — visuals alone don't drive adoption.
  • Start with 5 KPIs, not 25. Executives use dashboards more when there is less, not more.
  • Build the date table in Power Query, not DAX, for better performance at scale.
Future Enhancements
  • Predictive revenue forecasting using Azure Machine Learning integration
  • Customer churn prediction model via embedded Python visuals
  • Automated weekly KPI email snapshots via Power Automate
  • Mobile-optimized report layout for field sales team usage
  • Row-Level Security (RLS) for region-specific data access per manager
